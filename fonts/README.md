# CMDA Theme Fonts Directory

## Required Font Files

To complete the CMDA brand identity implementation, place the following **ALTA font files** in this directory:

### ALTA Font Files Needed:

- `ALTA-Light.woff2` & `ALTA-Light.woff` (Font-weight: 300)
- `ALTA-Regular.woff2` & `ALTA-Regular.woff` (Font-weight: 400)
- `ALTA-Medium.woff2` & `ALTA-Medium.woff` (Font-weight: 500)
- `ALTA-Bold.woff2` & `ALTA-Bold.woff` (Font-weight: 700)

## Where to Get ALTA Fonts

1. **Contact your design agency** or the creator of the CMDA visual identity chart
2. **Check brand assets repository** (shared drives, Dropbox, Google Drive)
3. **Font foundry/marketplace** if ALTA is a commercial font (MyFonts, Fonts.com, etc.)
4. **Request from CMDA communications team** if managing internal brand assets

## Important: Web Font License

⚠️ **Verify that your ALTA font license permits web embedding via @font-face**

Most commercial fonts require a separate web license. Ensure compliance before deployment.

## Font Format Priority

- `.woff2` - Modern browsers (Chrome, Firefox, Safari, Edge) - Best compression
- `.woff` - Fallback for older browsers - Universal support

## Current Setup

The theme is configured to use:

- **Primary Typography (Headings)**: ALTA → Montserrat (fallback)
- **Secondary Typography (Body)**: Montserrat (loaded via Google Fonts)

Until ALTA files are added, **Montserrat will display for all text**, maintaining brand consistency with the secondary typography.

## Testing After Upload

1. Upload all 8 ALTA files to this directory
2. Clear browser cache and WordPress cache
3. Visit your site and inspect headings (h1-h6)
4. Verify ALTA font loads in browser DevTools > Network tab

## Support

If you need help obtaining or installing the ALTA fonts, contact your WordPress developer or CMDA technical team.
